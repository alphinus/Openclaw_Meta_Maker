# Skill

## Name

/elvis-copywriting

## Beschreibung

Schreibt verkaufsfördernden Text nach dem PAS-Prinzip (Problem → Agitation → Solution) in 3 Formaten: Produkt-Post, Angebot-Tweet und DM-Pitch. Liefert einsatzbereite Texte mit expliziten Zeichenlimits je Format — optimiert für Konversion auf X/Twitter.

## Ziele

- 3 fertige Texte in den Formaten Produkt-Post, Angebot-Tweet und DM-Pitch
- Jeder Text folgt dem PAS-Prinzip und hält das Zeichenlimit des Formats ein
- Direkte, nutzenorientierte Sprache ohne generische Floskeln
- Je Format: Zeichenanzahl explizit dokumentiert

## Strategie

PAS-Copywriting funktioniert weil es dem natürlichen Entscheidungsweg folgt: zuerst das Problem benennen das der Leser kennt, dann den Schmerz verstärken damit die Relevanz klar ist, dann die Lösung präsentieren. Der Skill priorisiert konkrete Nutzensprache über Werbe-Floskeln — kein "günstig", "billig" oder "kostenlos" ohne konkreten Wert. Jedes Format hat ein eigenes Zeichenlimit das konsequent eingehalten wird.

## Einschränkungen

- Format 1 Produkt-Post: max. 280 Zeichen (P = 1 Satz, A = 1 Satz, S = 1 Satz + Preis)
- Format 2 Angebot-Tweet: max. 240 Zeichen (direkte Nutzensprache, 1 CTA am Ende)
- Format 3 DM-Pitch: 3 separate Nachrichten — Opener max. 150 Zeichen + Angebot max. 200 Zeichen + CTA max. 100 Zeichen
- Keine Wörter "günstig", "billig" oder "kostenlos" ohne konkreten Wert-Nachweis
- Kein automatisches Versenden — Output ist zur manuellen Freigabe bestimmt
- Genau 3 Formate pro Aufruf — nicht mehr, nicht weniger

## Ausführungsschritte

1. Lies das Produkt oder Angebot und identifiziere: (a) das konkrete Problem das gelöst wird (1 Satz), (b) den spezifischen Schmerzpunkt der Zielgruppe (1 Satz), (c) den Hauptnutzen der Lösung (1 Satz), (d) den Preis oder die Gegenleistung. Dokumentiere diese 4 Punkte als "PAS-Briefing" vor den Texten.
2. Schreibe Format 1 — Produkt-Post (max. 280 Zeichen): P (Problem, 1 Satz) + A (Agitation, 1 Satz, verstärkt den Schmerz) + S (Solution, 1 Satz + Preis). Alle 3 Elemente in einem Tweet. Zeichenanzahl in Klammern dokumentieren.
3. Schreibe Format 2 — Angebot-Tweet (max. 240 Zeichen): Direkte Nutzensprache, kein Umweg über das Problem — beginne sofort mit dem stärksten Nutzen. 1 CTA am Ende (z.B. "Link in Bio", "Jetzt buchen", "DM mir"). Zeichenanzahl dokumentieren.
4. Schreibe Format 3 — DM-Pitch (3 separate Nachrichten): Nachricht 1 = Opener (max. 150 Zeichen, persönliche Ansprache oder gemeinsame Referenz), Nachricht 2 = Angebot (max. 200 Zeichen, konkreter Nutzen + Preis), Nachricht 3 = CTA (max. 100 Zeichen, eine klare Handlung). Zeichenanzahl je Nachricht dokumentieren.
5. Prüfe alle 3 Formate auf verbotene Wörter: Falls "günstig", "billig" oder "kostenlos" ohne konkreten Wert-Nachweis vorkommt → markiere die Stelle mit "[WARNUNG: generischer Wert-Begriff — konkreten Nutzen oder Zahl ergänzen]" und schlage eine Alternative vor.

## Verifikation

- Alle 3 Formate vorhanden: Produkt-Post, Angebot-Tweet, DM-Pitch — je als eigener Block
- Format 1: ≤ 280 Zeichen, erkennbare PAS-Struktur (Problem, Agitation, Solution)
- Format 2: ≤ 240 Zeichen, endet mit CTA
- Format 3: 3 separate Nachrichten — Opener ≤ 150, Angebot ≤ 200, CTA ≤ 100 Zeichen
- Failure-Indikator: Text enthält "günstig", "billig" oder "kostenlos" ohne konkreten Wert → Markierung "[WARNUNG: generischer Wert-Begriff]" im Output
- Failure-Indikator: Kein Preis oder keine Gegenleistung im Briefing → Meldung "Preis fehlt — Format 1 und DM-Pitch unvollständig"
- Akzeptanzkriterium: 3 Formate innerhalb Zeichenlimits, PAS-Prinzip erkennbar, keine ungekennzeichneten generischen Wert-Begriffe

## Abhängigkeiten

- Input: Produkt-/Angebots-Beschreibung (1–5 Sätze), Zielgruppe, Preis oder Gegenleistung
- Empfohlene Vorgänger-Skills: /elvis-x-hook-writer (liefert starke Opener als Basis für Format 1 und 2)

## Output

PAS-Briefing (4 Punkte) + 3 Formate (je mit Zeichenanzahl in Klammern) + Warnmarkierungen falls vorhanden. Gesamtlänge: max. 500 Wörter, Markdown-Format.
